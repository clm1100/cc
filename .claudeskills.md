# commit

Analyze staged/unstaged changes and commit using Conventional Commits format.

## Steps

1. Run `git status` and `git diff HEAD` to understand all changes
2. Determine the appropriate type: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, or `chore`
3. Write a concise scope and description in English or Chinese (match the user's language)
4. Stage all changes with `git add -A`
